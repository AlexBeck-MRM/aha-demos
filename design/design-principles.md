# Design Principles

## 1. Lead with clarity
The user should understand what this page is for, what AHA can help with, and where to go next within the first screen.

## 2. Make utility feel designed
Search, navigation, tabs, cards, forms, and support modules should feel as art-directed as the hero.

## 3. Use tactility with restraint
Depth, rounded forms, and soft elevation should make the interface feel touchable without becoming gimmicky.

## 4. Keep red strategic
Red should feel meaningful and branded, not omnipresent.

## 5. Let humanity carry trust
Photography, illustration, and copy should make the site feel human before institutional.

## 6. Separate structure from expression
Shared information architecture can support multiple routes, but each route needs a distinct visual attitude.

## 7. Document decisions as you go
A direction is not stable until it is logged, compared, and connected back to the relevant route or inventory file.
