# Links And Guidance

## Direction
Links should read like clear next steps, not generic prompts. Use action-led link language, avoid redundant `Read more` labels on already clickable cards, and use icon cues to signal whether a link moves deeper into the site or out to another destination.

## Reasoning
- Explicit link language reduces ambiguity in journeys that already carry uncertainty.
- Removing duplicate prompts lowers noise and makes clickable surfaces feel cleaner.
- Icon cues build expectation and familiarity across the system.
- Guidance should explain why a path matters, not just where it goes.

## Guardrail
Avoid vague prompts like `Learn more` when a clearer phrase is available.
