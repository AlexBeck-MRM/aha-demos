# Image Types

## Live board status
Section is present on the workbench with header instance `2138:29796`, a left-to-right spectrum from `Functional` to `Expressive`, and five named image zones:

- Icons: `Utilitarian`
- Small illustrations: `Instructional`
- Large illustrations: `Representational`
- 3D objects: `Interactive`
- Photography: `Tangible`

The section also includes a lower example panel that suggests illustration itself may need internal sub-zones rather than being treated as one flat bucket.

## Direction
Use different image types for different jobs so the system stops feeling same-y. Icons, illustration, photography, and occasional 3D should each carry a distinct role rather than all trying to do the same thing.

## Reasoning
- Matching the medium to the message creates clearer hierarchy and faster understanding.
- A mix of image roles makes the system feel richer and more ownable without fragmenting it.
- Photography still carries lived proof and human credibility where trust matters most.
- 3D should stay occasional and only appear when direct inspection or manipulation adds real value.

## Working roles
- Read the board as a decision ladder: start with the user need, then choose the lightest image type that can do the job.
- `Icons` handle recognition and compact utility.
- `Small illustrations` explain, soften, and guide without the weight of photography.
- `Large illustrations` carry more editorial atmosphere and work best when paired with photography rather than replacing it.
- `3D objects` should be reserved for inspection, exploration, or manipulation.
- `Photography` carries lived proof, trust, and emotional credibility.

## Guardrail
Do not use image type as a substitute for route direction. Route expression still depends on treatment, placement, tone, and surrounding UI.

Do not let this board absorb decisions that belong in `Image cropping`, `Photography treatment`, `Illustration use`, or `Illustration Style`. This section should stay focused on role taxonomy.
