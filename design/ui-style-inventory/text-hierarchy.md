# Text Hierarchy

## Direction
Make hierarchy do the sorting work. Rely on a clear ladder from label to headline to deck to metadata to body, and use occasional crafted emphasis only when it adds warmth without confusing priority.

## Reasoning
- Dense health content needs structural cues before it needs decorative ones.
- Spacing, scale, weight, and placement should do most of the emphasis work.
- A small amount of authored highlighting can reintroduce warmth on article surfaces.
- We still need to keep testing the hierarchy on smaller screens and utility-heavy pages.

## Considerations
- Establish a stable ladder across modules: section label, headline, support copy, metadata, body, caption, and utility text.
- Let one text tier lead each module. If two tiers fight for attention, the module will feel noisy before any visual styling is added.
- Use spacing, placement, and type scale before color or inline emphasis. The board examples work because the hierarchy is structural, not decorative.
- Keep metadata readable but decisively subordinate. Dates, author lines, captions, and labels should help orientation without stealing focus from the main message.
- Reserve the most expressive type moments for headlines or key mission statements. Body and utility copy should stay calmer and more repeatable.
- Make sure article, award, and mission-style modules can all use the same hierarchy logic without feeling flattened into one generic text treatment.
- Preserve the order of importance on mobile. Wrapping and stacking can change the shape of a module, but not the reading priority.
- Protect accessibility basics as part of hierarchy, not after it: line length, paragraph spacing, contrast, and clearly distinct link states.

## Open questions
- Where should larger body styles be used to improve spaciousness on low-copy modules?
- How much serif or display contrast can we introduce in hero moments before the system starts to split from utility content?
