# Accessibility

## Direction
Treat accessibility as structural, not decorative. Contrast, hierarchy, keyboard logic, language access, scalable type, and motion restraint should be designed into the core system from the start. Preference controls can help, but the default experience still needs to be clear and usable without extra effort.

## Reasoning
- AHA serves broad audiences in moments that can already feel stressful or cognitively heavy.
- Clear type, stable layouts, and obvious focus states improve usability for everyone, not only people using assistive tools.
- Language and reading support are especially important in a health context where comprehension is part of trust.
- Accessibility controls should add agency without disrupting the first view or fragmenting the main journey.

## Guardrail
Every expressive decision should survive an accessibility check.
