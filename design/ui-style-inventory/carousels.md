# Carousels

## Direction
Use carousels only when a grouped content set genuinely benefits from browsing side to side. Keep the first frame meaningful, allow natural swipe or drag behavior, and always provide clear fallback button controls.

## Reasoning
- Carousels can make grouped content feel richer without forcing a long vertical stack.
- Swipe plus explicit controls creates a more predictable and accessible interaction pattern.
- The first visible frame needs to stand on its own because hidden content is always weaker.
- Overuse quickly turns a useful browse pattern into a content-hiding habit.

## Open questions
Should carousel usage be limited to campaigns and content collections rather than core task pathways?
