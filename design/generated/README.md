# Generated Assets

## Job
- Keep generated visual work that should remain findable.
- Keep `output/` disposable.
- Separate review assets from accepted assets.

## Destinations
- Review candidates: `design/generated/review/<yyyy-mm-dd>-<topic>/`.
- Accepted generated images: `design/generated/<topic-or-system>/`.
- Accepted icon sets: `design/generated/icons/<style-or-set>/`.
- Production-ready reusable brand assets: `reference/assets/`.

## Rules
- Use `output/` for local scratch only.
- Copy review-worthy generated assets out of `output/` before running clean commands.
- Add a `manifest.md` to each generated review folder.
- Add durable generated asset folders to `projects/aha-website-refresh/artifact-index.yaml`.
- Add current generated asset locations to `projects/aha-website-refresh/WHERE-THINGS-LIVE.md`.
- Do not mark a review folder as accepted direction without an explicit design decision.
