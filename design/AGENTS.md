# Design Guidance

Treat `design/` as the working memory for direction, not a moodboard dump.
Update route docs and the relevant UI inventory file whenever a design stance changes.
Keep Route A and Route B distinct unless an explicit decision collapses them.
When a task references the Figma workbench, use `design/figma/workbench-section-map.yaml` to resolve the exact section and editable header text paths before inspecting broader board metadata.
