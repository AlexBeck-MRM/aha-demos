# CTA Logic

## CTA hierarchy
- Primary CTA: immediate action with concrete value.
- Secondary CTA: broader exploration or lower-friction support pathway.
- Tertiary CTA: utility or proof detail.

## CTA rule
Every major module should make the next action legible. Avoid generic CTA reuse when the destination can be named more clearly.

## Route difference
Route A can support slightly more human or editorial CTA framing. Route B should stay especially crisp and literal.
